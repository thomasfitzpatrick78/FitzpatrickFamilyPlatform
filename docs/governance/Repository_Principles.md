# Repository Principles

**Document Version:** 1.0

**Status:** Active

**Milestone:** Milestone 11

---

## Purpose

This document defines principles for preserving Platform knowledge, governance, architecture, and engineering work.

---

## Principles

- The repository is the authoritative source of truth.
- Conversations are transient working sessions.
- Governance is version controlled.
- Architecture decisions must be documented.
- Product ownership is explicit before implementation.
- Deterministic validation should be automated.
- Evidence is preferred over assertion.
- Platform scope excludes finance, banking, budgeting, transactions, and investments.
- Finance remains in the Fitzpatrick Family Financial Assistant repository.
- Cross-repository relationships are coordinated through portfolio governance, not shared implementation by default.
- Shared engineering patterns may be reused, but shared implementation requires future architecture approval.
