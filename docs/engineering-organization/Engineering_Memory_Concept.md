# Engineering Memory Concept

**Document Version:** 1.0

**Status:** Active

**Milestone:** EO-13.0

---

## Purpose

This document defines Engineering Memory for the AI Engineering Organization.

---

## Concept

Engineering Memory is the durable, reviewable, git-managed record of product intent, architecture decisions, delivery evidence, registry state, and learned operating practices.

Chats are useful working sessions. Repository artifacts are durable Engineering Memory.

---

## Memory Sources

| Source | Memory Type |
|--------|-------------|
| Governance documents | Operating rules, roles, principles, and ownership. |
| Product documents | Vision, capabilities, backlog, roadmap, and customer value. |
| Architecture documents | ADRs, architecture backlog, models, and patterns. |
| Requirements and specifications | Approved scope and expected behavior. |
| Registries | Authoritative structured operational state. |
| Reports | Validation output and release evidence. |
| Milestone artifacts | Delivery history and closure evidence. |

---

## Memory Rules

- Durable knowledge belongs in the repository.
- Historical decisions are preserved rather than rewritten.
- Evidence is preferred over opinion.
- Repeated practices are evaluated before promotion.
- Memory must remain reviewable by humans and usable by AI delivery roles.
