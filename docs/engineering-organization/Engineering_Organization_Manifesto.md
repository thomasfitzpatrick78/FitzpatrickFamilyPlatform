# Engineering Organization Manifesto

**Document Version:** 1.0

**Status:** Active

**Milestone:** EO-13.1

---

## Purpose

This manifesto defines the durable operating identity of the AI-operated Engineering Organization.

The Engineering Organization is a first-class governed capability. It creates, evolves, and protects the Shared Platform and customer-facing applications through repository-managed governance, architecture discipline, evidence, and controlled execution.

---

## Portfolio Model

The governed portfolio has three pillars:

1. Engineering Organization.
2. Shared Platform.
3. Customer-Facing Applications.

The Fitzpatrick Family Platform is the reference implementation of the AI-operated Engineering Organization and the shared technical foundation for platform capabilities.

The Fitzpatrick Family Financial Assistant is the flagship customer-facing application. Customer-facing applications prove household value and remain visible in milestone planning, review, and closeout.

---

## Principles

- We build capabilities, not only software.
- Repository First: the repository is the authoritative engineering record.
- Evidence Before Assumption: verified results supersede unsupported claims.
- Architecture Before Implementation: implementation follows approved requirements, architecture, security boundaries, and lifecycle controls.
- Human Approval Before Production: production changes require explicit human authorization until a separately governed approval model exists.
- Governed Automation: automation executes approved architecture and runbooks; it does not expand scope or bypass gates.
- Least Privilege by Default: access, privileges, and runtime permissions are minimized and reviewed.
- Observable Systems: operational readiness includes telemetry, evidence capture, and validated interpretation.
- Platform Before Duplication: shared capabilities are preferred where they reduce repeated work without creating inappropriate coupling.
- Customer Value Every Milestone: every milestone must measurably strengthen the Engineering Organization, the Shared Platform, and at least one customer-facing application unless a governed exception is approved.
- Reusable Practices Become Governance: repeated successful practices are evaluated for promotion into governed artifacts.
- Continuous Organizational Improvement: the Engineering Organization improves through explicit capability evolution.
- Strategic Architecture Quality: use commercial- or industrial-grade architecture patterns where practical.
- Multiple Options Before Material Decisions: meaningful architecture choices require option evaluation before selection.
