## Purpose

This template defines the required transition package from one governed milestone to the next.

## Required Sections

### Approved Vision

- Portfolio intent.
- Milestone intent.
- EO, PLAT, and customer-facing application alignment.

### Current-State Baseline

- Repository state.
- Release state.
- Runtime or operational state where applicable.
- Registry and Digital Twin state.
- Known current-state versus planned-state distinctions.

### Unresolved Risks

- Risk.
- Owner.
- Stop condition.
- Required decision.
- Target review point.
